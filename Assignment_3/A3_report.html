Reading & Critique

Insights about the data/visualization:

  • From 2020 to 2022, the rate of new COVID-19 cases was highest at the end of this range, in January of 2022.
  • The rate of new COVID-19 cases seems to increase the most during the winter, then decrease later.
  • New COVID_19 cases are generally becoming more and more frequent as time goes on.

Design Critique:

  • The graph uses a spiral to display data along the axis of time, such that data with the same date and different years are shown at the same angle, making it easy to compare information within a particular month, as well as track the data over several months. It's easy to see where new COVID-19 cases are at their highest, and where fluctuations repeat throughout the year.
  • The arc length on the time axis representing a month gets longer as the graph progresses, having a similar effect to using an exponential scale for the time axis. This does not seem to have any purpose in the graph, and could mislead viewers into misinterpreting the rate of increase of COVID-19 cases.
  • The graph uses minimal colors to indicate the data, and just enough labels are used to make the graph readable. It's also straightforward to find the start of the graph and read it through to the end. It would be nice if certain important events related to the pandemic were indicated at certain parts of the time axis, so inferences could be drawn about what caused certain fluctuations in COVID-19 rates.

Visualization Sketches

Design Rationale for Sketch 1:

  • Since the time axis always remains the same distance from the center of the graph, the arc length for each month is now the same for every year in which there is data, removing any unwanted visual biases and making the data easier to read and interpret.
  • Color would be used to indicate the year, instead of height. This graph would use a gradient on which it's easier to locate where certain colors are. But maybe the constant change in color would make it harder to differentiate between the years...
  • The month labels are moved in towards the center, to give room to the graph data. They don't have to be rotated like they are in the sketch, as long as they are clearly visible.

Design Rationale for Sketch 2:

  • This graph intends to make data easier to read by removing the circle shape of the original graph entirely, opting for sub-graphs that can be read like a typical piece of text. It's less visually interesting and unique, but I suppose not every graph has to be a work of art to get its point across.
  • Separating the data by year into their own sub-graphs makes it easier to differentiate between each set of data, without the viewer having to check a legend to keep track of the years. As such, there is no longer any color in this graph.
  • In fact, this graph has no legend at all. All of the labels are now embedded into the graph itself, like how the vertical axis shows the height of certain new case rates instead of the triangular width indicator from the original graph. I would've moved the year labels to the other side of the sub-graphs for clarity, but there were explicit instructions to write in pen. I don't want to find out what would happen if I wrote in pencil instead...

Design Rationale for Sketch 3:

  • The intention with this sketch was to simplify the display further by putting all of the data on the same set of axes, making comparisons between years even simpler.
  • The axes and labels are left unchanged from the last sketch, except that the three lines are now labeled with their respective years. It might be a bit harder to differentiate between the years now, so maybe I would add color back in, for a layer of redundancy.
  • While there are ticks for every month along the horizontal axis, The month labels only include January, April, July, and October. This was an arbitrary choice borrowed from the original graph, but I realize there isn't really any point in leaving out the other months, besides the fact that they might not all fit. We'll see how this axis looks in the final graph.
